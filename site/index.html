Current Real-Data Control

Text-to-profile retrieval is now measured against stricter baselines.

The full metadata baseline is intentionally easy because it includes direct perturbation identifiers. The identifier-stripped baseline removes those direct identifiers from candidate text and is the more honest control for future model work.

Mode mAP Hit@1 Hit@5 Hit@10
Full metadata TF-IDF 0.9952 0.9906 0.9938 0.9984
Identifier-stripped TF-IDF 0.3991 0.2496 0.4774 0.5835
Random 0.0032 0.0000 0.0078 0.0156
Shuffled-label 0.0017 0.0000 0.0062 0.0156

Scientific Guardrails

What the project can say right now.

Validated

Real local JUMP profile files can be audited, indexed, queried, and reported with controls.

Measured

Same-treatment retrieval is compared with random and shuffled-label baselines.

Limited

Current local data has one inferred batch, so batch generalization is not yet proven.

Not claimed

This is not yet biological image understanding or final natural-language retrieval.

Phase 3 Gate

What must be true before stronger model claims.

  • Run the same benchmark on broader batches or datasets.
  • Beat identifier-stripped metadata controls, not only random baselines.
  • Report held-out plate, batch, well, and perturbation behavior.
  • Keep raw data, generated reports, embeddings, and indexes outside git.

Reproducibility Path

Run the safe baseline workflow locally.

  1. python scripts/run_phase2_jump_smoke.py
  2. python scripts/audit_jump_pilot.py --summary-only --max-columns-to-print 20
  3. python scripts/build_jump_profile_index.py
  4. python scripts/run_jump_profile_diagnostics.py --filtered-presets
  5. python scripts/run_jump_text_profile_retrieval.py
  6. python scripts/make_phase2_jump_report.py --out outputs/jump_pilot_phase2_report.md

Data, embeddings, indexes, model weights, raw images, and generated outputs stay out of git.
